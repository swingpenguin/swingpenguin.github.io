How to change theme in Hexo and deploy to Github pages

Created2021-05-04|Updated2021-05-12|tutorial
|Word count:417|Reading time:2min|Post View:

Install Hexo

Requirements

First, we need to have Node.js and Git installing.

Step 1: Install Hexo globally

$ npm install -g hexo-cli

Step 2: Init Hexo

Init the blog folder and run npm install.

Change Theme

The default theme of Hexo framework is landscape. It’s not bad but we always want a better one, right?
Hexo provides more than 300 themes on their website, most of them are developed by users and able to clone from GitHub.

Step 1: Put new theme under themes folder

themes folder is under the your_blog_folder we just inited.
Take my blog as an example. I use the theme Butterfly.
So I run to clone butterfly from the theme repo to my themes folder.
$ git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

Step 2: Update _config.yml file

There are two _config.yml files, the one we need to update is under the blog folder, not the themes folder.
Default:
theme: landscape
Change to:
theme: new_theme_name
In this case, it’s
theme: butterfly

Deploy to GitHub pages

There are two ways to deploy our Hexo blog to GitHub. The first way refers to this link. In this way, the source folder, which contains the original files of our blog, especially markdown files of posts, will be uploaded to Github. So I prefer the second way, which will not upload the source folder to GitHub, and it’s called One-command deployment.

Step 1: Create a github repository for deploying blog

The preferred name is username.github.io.

Step 2: Install hexo-deployer-git

$ npm install hexo-deployer-git

Step 3: Update _config.yml file

We need to update the configuration file _config.yml which is under the blog folder. Add sentences below:

Here the branch could be any branch we want to deploy our website to. As the repo is only for the website, the master branch is a good choice because it’s the default branch. However, we can still use other branches like gh-pages.

Step 4: Run hexo clean && hexo deploy

Just run
$ Hexo clean
then
$ Hexo deploy

Then we’ve done!

Update the website

Just run
$ Hexo generate or $ Hexo g
then
$ Hexo deploy

Then, check your website! It may take a few seconds to update!

Custom domain

Put CNAME file under ./source folder.
By doing this, CNAME will be compiled every time deploying.
